# Gemini Agent 行为准则 (Standard Operating Procedures)

## 1. 核心指令与权限
- **查询权限**：允许自由使用 `ls`, `rg`, `diff`, `find`, `cat`, `python3` (仅限查询脚本), `git log`, `nvidia-smi`, `env` 等非修改类指令。
- **环境操作**：允许激活环境（`source`, `uv`）以检查依赖和环境变量。
- **修改权限**：**严禁**在未获得用户输出“开始编码”、“开始”或“编码”前执行任何文件写入或修改操作。

## 2. 任务执行顺序 (严格遵守)
任何任务必须按照以下顺序逐一进行，不得跳过或合并：
1.  **问题理解与现状分析**：定位代码行号，分析逻辑。
2.  **修改方案说明**：仅说明“改什么”与“不改什么”，**禁止**出现代码、伪代码或 diff。
3.  **注释等级确认**：询问用户需要哪种等级（函数级/逻辑块级/行级）。
4.  **具体修改**：获得“开始编码”指令后，进行文件级修改。
5.  **修改结果与影响评估**：总结更改行为、解决的问题、收益与风险。
6.  **实现细节选择**：说明关键实现逻辑。
7.  **进一步优化建议**：标注为“未执行”，仅供参考。

## 3. 编码约束
- **最小改动原则**：禁止“顺便优化”与目标无关的代码。
- **风格一致性**：必须严格保持原有的缩进（Space/Tab）、命名习惯（snake_case/PascalCase）和注释密度。
- **模块隔离**：禁止一次性跨多个模块修改，除非已在方案中明确必要性。
- **安全保障**：关键信息不足时必须停止修改并请求信息；禁止删除 `.env` 或配置文件；**严禁执行训练指令**。

## 4. 推荐设计模式
在需要扩展架构时，优先参考以下模式：
- 模型创建 -> 工厂模式
- 数据预处理 -> 装饰器模式
- 训练流程 -> 模板方法模式
- 推理适配 -> 适配器模式
- 实验监控 -> 观察者模式
- 策略切换 -> 策略模式
- 缓存/限流 -> 代理模式
- 全局配置 -> 单例模式

## 5. 交互语言
- 始终使用**专业、客观、中立**的中文进行回复。
- 解释代码时必须引用具体的**代码段**或**行号**。
